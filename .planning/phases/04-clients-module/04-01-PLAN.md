# Plan 04-01: Clients Module

## Goal

Add a "Clientes" section to Pyrux OS that shows prospects with `status === "cerrado"` in a dedicated route with a simplified table. Prospects with status "cerrado" are filtered OUT of the Prospectos views. Clicking a client row opens the existing ProspectSheet. A "Proyectos" column shows a badge with the count of linked projects.

No database changes. Pure client-side filtering on the shared React Query cache.

## Must Haves

1. "Clientes" appears in the sidebar after "Prospectos"
2. `/clientes` route renders the clients table
3. Prospects with `status === "cerrado"` appear ONLY in Clientes, NOT in Prospectos
4. Clicking a client row opens ProspectSheet (reused, not duplicated)
5. "Proyectos" column shows a numeric badge with count of projects linked via `prospect_id`
6. Changing status away from "cerrado" in the sheet moves the record back to Prospectos (automatic via shared cache)

### Task 4: Create the ClientesShell component

**File (new):** `components/modules/clientes/clientes-shell.tsx`

**What:** Shell component that fetches data, manages sheet state, and renders the client table.

**How:**
```tsx
"use client";

import { useState, useMemo } from "react";
import { ClientTable } from "./client-table";
import { ProspectSheet } from "@/components/modules/prospectos/prospect-sheet";
import { useProspects } from "@/hooks/use-prospects";
import { useProjects } from "@/hooks/use-projects";
import { usePagination } from "@/hooks/use-pagination";
import { Button } from "@/components/ui/button";
import type { Prospect } from "@/lib/types/database.types";

export function ClientesShell() {
  const [sheetOpen, setSheetOpen] = useState(false);
  const [editingProspect, setEditingProspect] = useState<Prospect | null>(null);

  const { data: allProspects = [], isLoading: loadingProspects } = useProspects();
  const { data: projects = [], isLoading: loadingProjects } = useProjects();

  // Only prospects with status "cerrado" are clients
  const clients = useMemo(
    () => allProspects.filter((p) => p.status === "cerrado"),
    [allProspects]
  );

  // Count projects per prospect_id
  const projectCounts = useMemo(() => {
    return projects.reduce<Record<string, number>>((acc, project) => {
      if (project.prospect_id) {
        acc[project.prospect_id] = (acc[project.prospect_id] || 0) + 1;
      }
      return acc;
    }, {});
  }, [projects]);

  const { visibleItems, hasMore, remaining, loadMore } = usePagination(clients);

  function handleEdit(prospect: Prospect) {
    setEditingProspect(prospect);
    setSheetOpen(true);
  }

  function handleSheetOpenChange(open: boolean) {
    setSheetOpen(open);
    if (!open) setEditingProspect(null);
  }

  return (
    <div className="space-y-4">
      <div>
        <h1 className="text-2xl font-semibold text-foreground">Clientes</h1>
        <p className="text-sm text-muted-foreground mt-0.5">
          Prospectos cerrados — clientes activos
        </p>
      </div>

      <ClientTable
        clients={visibleItems}
        projectCounts={projectCounts}
        isLoading={loadingProspects || loadingProjects}
        onEdit={handleEdit}
      />

      {hasMore && (
        <div className="flex justify-center pt-2">
          <Button variant="outline" size="sm" onClick={loadMore}>
            Cargar mas ({remaining} restantes)
          </Button>
        </div>
      )}

      <ProspectSheet
        open={sheetOpen}
        onOpenChange={handleSheetOpenChange}
        prospect={editingProspect}
      />
    </div>
  );
}
```

Key points:
- Reuses `useProspects()` (shared React Query cache — same `["prospects"]` key)
- Reuses `ProspectSheet` directly (no wrapper, no modifications)
- Reuses `usePagination` for load-more behavior
- `projectCounts` is a `Record<string, number>` mapping prospect ID to project count
- No "Nuevo" button here — clients are created by setting a prospect's status to "cerrado"

### Task 5: Create the ClientTable component

**File (new):** `components/modules/clientes/client-table.tsx`

**What:** Simplified table for clients. Columns: Nombre, Empresa, Email, Telefono, Proyectos (badge count), edit button. Row click opens the sheet.

**How:**
```tsx
"use client";

import {
  Table,
  TableBody,
  TableCell,
  TableHead,
  TableHeader,
  TableRow,
} from "@/components/ui/table";
import { Badge } from "@/components/ui/badge";
import { Skeleton } from "@/components/ui/skeleton";
import { EmptyState } from "@/components/shared/empty-state";
import { UserCheck } from "lucide-react";
import type { Prospect } from "@/lib/types/database.types";

interface ClientTableProps {
  clients: Prospect[];
  projectCounts: Record<string, number>;
  isLoading: boolean;
  onEdit: (prospect: Prospect) => void;
}

export function ClientTable({
  clients,
  projectCounts,
  isLoading,
  onEdit,
}: ClientTableProps) {
  if (isLoading) {
    return (
      <div className="space-y-2">
        {Array.from({ length: 6 }).map((_, i) => (
          <Skeleton key={i} className="h-12 w-full rounded-md" />
        ))}
      </div>
    );
  }

  if (!clients.length) {
    return (
      <EmptyState
        icon={UserCheck}
        title="Sin clientes todavia"
        description="Cuando un prospecto pase a estado 'Cerrado', aparecera aca."
      />
    );
  }

  return (
    <div className="rounded-md border border-border overflow-hidden">
      <Table>
        <TableHeader>
          <TableRow className="hover:bg-transparent">
            <TableHead>Nombre</TableHead>
            <TableHead>Empresa</TableHead>
            <TableHead>Email</TableHead>
            <TableHead>Telefono</TableHead>
            <TableHead>Proyectos</TableHead>
          </TableRow>
        </TableHeader>
        <TableBody>
          {clients.map((client) => {
            const count = projectCounts[client.id] ?? 0;
            return (
              <TableRow
                key={client.id}
                className="hover:bg-secondary/50 cursor-pointer"
                onClick={() => onEdit(client)}
              >
                <TableCell className="font-medium">{client.name}</TableCell>
                <TableCell className="text-muted-foreground text-sm">
                  {client.business ?? "\u2014"}
                </TableCell>
                <TableCell className="text-sm text-muted-foreground">
                  {client.email ?? "\u2014"}
                </TableCell>
                <TableCell className="text-sm text-muted-foreground">
                  {client.phone ?? "\u2014"}
                </TableCell>
                <TableCell>
                  <Badge variant={count > 0 ? "default" : "secondary"} className="text-xs">
                    {count}
                  </Badge>
                </TableCell>
              </TableRow>
            );
          })}
        </TableBody>
      </Table>
    </div>
  );
}
```

Key points:
- Entire row is clickable (`onClick` on `<TableRow>`, `cursor-pointer` class)
- No edit button icon — the whole row triggers the sheet (cleaner UX for a read-heavy view)
- Proyectos column: `Badge` with variant "default" if count > 0, "secondary" if 0
- Simplified columns vs ProspectTable: no Sector, Estado, Fuente, Asignado, Ultimo contacto
- Uses `"\u2014"` (em dash) for null values, matching ProspectTable pattern

## Verification

After implementing all tasks, verify each success criterion:

1. **Sidebar:** Navigate to the app. "Clientes" should appear between "Prospectos" and "Proyectos" in the sidebar with the UserCheck icon.

2. **Route works:** Click "Clientes" in the sidebar. The page should load at `/clientes` showing the client table.

3. **Filtering is correct:**
   - Create or find a prospect with `status = "cerrado"`.
   - It should appear in `/clientes` but NOT in `/prospectos` (not in table, kanban, or pipeline).
   - A prospect with any other status should appear in `/prospectos` but NOT in `/clientes`.

4. **Row click opens sheet:** Click any row in the Clientes table. The ProspectSheet should open showing that client's data with all fields editable.

5. **Proyectos badge:** If a client has projects linked via `prospect_id`, the Proyectos column shows the correct count in a badge. If no projects, shows "0" with a muted badge.

6. **Status change moves record:** Open a client's sheet, change status from "cerrado" to any other value (e.g., "en_negociacion"), save. The record should disappear from Clientes and reappear in Prospectos — without page refresh (React Query cache invalidation handles this automatically).
