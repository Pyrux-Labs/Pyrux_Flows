# Plan 03-01: Undo Delete

## Goal

Replace the ConfirmDialog-based delete flow across all 5 modules with a deferred-delete pattern: clicking "Eliminar" immediately closes the sheet, removes the item from the UI optimistically, and shows a Sonner toast with an "Deshacer" button. If the user does not undo within 5 seconds, the server delete executes. If they click "Deshacer", the timeout is cancelled and the item is restored to the query cache.

## Must Haves

1. ConfirmDialog is removed from all 5 sheets (no more confirmation modal on delete)
2. Toast shows a clear message with the item's display name
3. Clicking "Deshacer" within 5 seconds cancels the server delete and restores the item in the UI
4. After 5 seconds without undo, the server delete executes
5. If the server delete fails, a toast.error is shown and the item is restored in the cache

## Tasks

### Task 1: Create `useDeleteWithUndo` hook

**File:** `hooks/use-delete-with-undo.ts`

**What:** Create a reusable hook that encapsulates the entire deferred-delete + undo pattern. This hook does NOT call `useMutation` itself -- it receives the mutation's `mutateAsync` from the calling sheet, and handles the cache manipulation directly via `useQueryClient`.

**How:**

```ts
import { useRef, useCallback } from "react";
import { useQueryClient, type QueryKey } from "@tanstack/react-query";
import { toast } from "sonner";

interface DeleteWithUndoOptions {
  // The mutation's mutateAsync function (from the existing useDelete* hook)
  mutateAsync: (id: string) => Promise<unknown>;
  // Query key for simple list queries (e.g. ["prospects"])
  queryKey?: QueryKey;
  // Base key for filtered queries (e.g. ["expenses"]) -- used with setQueriesData
  baseKey?: string[];
}

interface TriggerOptions {
  id: string;
  label: string; // Display name for the toast (e.g. prospect.name)
}

export function useDeleteWithUndo({ mutateAsync, queryKey, baseKey }: DeleteWithUndoOptions) {
  const queryClient = useQueryClient();
  const timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);

  const handleDelete = useCallback(
    ({ id, label }: TriggerOptions) => {
      // Step 1: Snapshot the current cache for rollback
      let snapshot: unknown;

      if (baseKey) {
        // Filtered queries (expenses, income) -- snapshot all cache entries
        const entries = queryClient.getQueriesData<{ id: string }[]>({ queryKey: baseKey });
        snapshot = entries;
        // Optimistically remove from all matching cache entries
        queryClient.setQueriesData<{ id: string }[]>(
          { queryKey: baseKey },
          (old = []) => old.filter((item) => item.id !== id),
        );
      } else if (queryKey) {
        // Simple list queries (prospects, projects, services)
        snapshot = queryClient.getQueryData(queryKey);
        queryClient.setQueryData<{ id: string }[]>(queryKey, (old = []) =>
          old.filter((item) => item.id !== id),
        );
      }

      // Step 2: Schedule the actual server delete after 5 seconds
      timeoutRef.current = setTimeout(async () => {
        try {
          await mutateAsync(id);
          // After server confirms, invalidate to sync
          if (baseKey) {
            queryClient.invalidateQueries({ queryKey: baseKey });
          } else if (queryKey) {
            queryClient.invalidateQueries({ queryKey });
          }
        } catch {
          // Server delete failed -- restore from snapshot
          restoreSnapshot(snapshot);
          toast.error(`Error al eliminar "${label}"`);
        }
      }, 5000);

      // Step 3: Show toast with undo action
      toast(`"${label}" eliminado`, {
        duration: 5000,
        action: {
          label: "Deshacer",
          onClick: () => {
            if (timeoutRef.current) {
              clearTimeout(timeoutRef.current);
              timeoutRef.current = null;
            }
            restoreSnapshot(snapshot);
          },
        },
      });

      // Helper to restore snapshot
      function restoreSnapshot(snap: unknown) {
        if (baseKey && Array.isArray(snap)) {
          // Restore all filtered query entries
          (snap as [QueryKey, unknown][]).forEach(([key, data]) => {
            queryClient.setQueryData(key, data);
          });
        } else if (queryKey && snap !== undefined) {
          queryClient.setQueryData(queryKey, snap);
        }
      }
    },
    [mutateAsync, queryKey, baseKey, queryClient],
  );

  return { handleDelete };
}
```

Key design decisions:
- The hook receives `mutateAsync` from the existing `useDelete*` hooks, so we do NOT duplicate mutation logic or Supabase calls.
- It bypasses the optimistic handlers in `simpleDeleteHandlers`/`filteredDeleteHandlers` because we manually manage the cache (the mutation is called after the 5s delay, not immediately).
- It supports both `queryKey` (simple list: prospects, projects, services) and `baseKey` (filtered: expenses, income).
- The `mutateAsync` call still triggers `onSettled` invalidation from the existing mutation hooks, which is fine (double-invalidate is harmless, ensures sync).

**Verify:** File compiles without errors: `npx tsc --noEmit hooks/use-delete-with-undo.ts` or run the full `npm run build`.

For **income-sheet.tsx** (filtered query -- uses baseKey):


**c) Delete the `onDelete` async function entirely** (the one with try/catch/mutateAsync).

**d) Replace the ConfirmDialog JSX block** in SheetFooter with a plain Button:

```tsx
{isEditing && (
  <Button
    type="button"
    variant="destructive"
    size="sm"
    onClick={() => {
      // Close sheet first, then trigger deferred delete
      onOpenChange(false);
      handleDelete({
        id: item.id,       // prospect.id / project.id / expense.id / income.id / service.id
        label: item.name,  // or item.description for expenses/income
      });
    }}
  >
    Eliminar
  </Button>
)}
```

The `label` field varies per module:
- **prospect-sheet.tsx:** `prospect.name`
- **project-sheet.tsx:** `project.name`
- **expense-sheet.tsx:** `expense.description`
- **income-sheet.tsx:** `income.description`
- **service-sheet.tsx:** `service.name`

**e) Clean up unused imports.** After removing ConfirmDialog, also remove the `Loader2` import IF it was only used for the delete button spinner (check: it is also used for the submit button spinner, so keep it). Remove the ConfirmDialog import line.
