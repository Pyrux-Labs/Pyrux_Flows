# Plan 01-01: UX Interactions

## Goal

Make every data row/card clickable to open the edit sheet, and add a global Cmd+N / Ctrl+N shortcut to open the creation sheet from any module. Two small UX wins that make the app feel more responsive and professional.

## Must Haves

1. Click on any cell of a row in prospectos, proyectos, gastos, finanzas opens the edit sheet for that record
2. Click on a service card in tarifas opens the edit sheet for that service
3. The pencil icon button still works in all 5 modules (no regression)
4. Interactive elements inside rows (checkboxes in finanzas, switch in tarifas cards) do NOT trigger the edit sheet
5. Cmd+N / Ctrl+N from prospectos, proyectos, gastos, finanzas, or tarifas opens the creation sheet (empty form)
6. Cmd+N / Ctrl+N from Dashboard does nothing (no shell registers the shortcut)
7. The shortcut does NOT fire when the user is typing in an input/textarea

### Task 3: Create `useCreateShortcut` hook

**File (new):** `hooks/use-create-shortcut.ts`

Create a hook modeled after `hooks/use-command-palette.ts` that listens for Cmd+N / Ctrl+N and calls a callback.

```typescript
"use client";

import { useEffect } from "react";

/**
 * Registers a global Cmd+N / Ctrl+N keyboard shortcut.
 * Calls `onCreate` when triggered, unless the user is focused
 * on an input, textarea, or contentEditable element.
 */
export function useCreateShortcut(onCreate: () => void) {
  useEffect(() => {
    function handleKeyDown(e: KeyboardEvent) {
      // Skip if user is typing in a form field
      const tag = (e.target as HTMLElement)?.tagName;
      if (tag === "INPUT" || tag === "TEXTAREA" || (e.target as HTMLElement)?.isContentEditable) {
        return;
      }

      if ((e.metaKey || e.ctrlKey) && e.key === "n") {
        e.preventDefault();
        onCreate();
      }
    }
    document.addEventListener("keydown", handleKeyDown);
    return () => document.removeEventListener("keydown", handleKeyDown);
  }, [onCreate]);
}
```

Key details:
- `e.preventDefault()` stops the browser from opening a new window/tab
- The `INPUT`/`TEXTAREA`/`contentEditable` guard prevents firing while typing in a form (e.g., inside the sheet itself)
- The `onCreate` callback should be wrapped in `useCallback` at the call site (or be a stable function reference like the `handleAdd` functions defined with `function` keyword inside components)

### Task 4: Wire `useCreateShortcut` into all 5 module shells

**Files:**

- `components/modules/prospectos/prospectos-shell.tsx`
- `components/modules/proyectos/proyectos-shell.tsx`
- `components/modules/gastos/gastos-shell.tsx`
- `components/modules/finanzas/finanzas-shell.tsx`
- `components/modules/tarifas/service-grid.tsx` (tarifas has no separate shell -- ServiceGrid IS the shell)

**Changes per file (identical pattern):**

1. Add import at top:
```tsx
import { useCreateShortcut } from "@/hooks/use-create-shortcut";
```

2. Call the hook inside the component, after the `handleAdd` function definition:
```tsx
useCreateShortcut(handleAdd);
```

That is it. The `handleAdd` function is defined with the `function` keyword in each shell, so it is hoisted and stable enough for the effect. Since the effect re-registers on every render anyway (the `onCreate` dep changes), this is fine -- the listener is lightweight.

**No changes needed in Dashboard.** Since Dashboard does not render any of these shells, no `useCreateShortcut` is registered, so Cmd+N simply does nothing on Dashboard. This matches the requirement.
